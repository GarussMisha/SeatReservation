# 🔧 Исправление времени в уведомлениях

## 📅 Дата: 1 апреля 2026

## 📊 Проблема

Уведомления отображались с неверным временем:
- Отмена бронирования 5-10 минут назад
- Отображалось как "5 часов назад"

---

## 🔍 Причина

**Сервер (Backend):**
- Сохранял время в UTC через `datetime.utcnow()`
- Возвращал время в формате ISO **без указания часового пояса**
- Пример: `2026-04-01T10:00:00.123456` (неясно, UTC это или локальное время)

**Клиент (Frontend):**
- Получал время без указания часового пояса
- Браузер интерпретировал время как **локальное**
- При вычислении разницы (`now - date`) получалась разница в 5-7 часов (в зависимости от часового пояса пользователя)

---

## ✅ Решение

### 1. Backend — добавлен суффикс 'Z' (UTC)

**Файл:** `app/api/v1/routes/notification.py`

```python
# БЫЛО (неправильно):
"created_at": notification.created_at.isoformat() if notification.created_at else None,

# СТАЛО (правильно):
"created_at": notification.created_at.isoformat() + 'Z' if notification.created_at else None,
```

**Результат:** `2026-04-01T10:00:00.123456Z` — явно указано, что время UTC

---

### 2. Frontend — корректная обработка UTC

**Файлы:**
- `src/views/Notifications.vue`
- `src/components/NotificationsDropdown.vue`

**Изменения:**

```javascript
// ДОБАВЛЕНА проверка валидности даты
if (isNaN(date.getTime())) {
  console.error('Неверный формат даты:', dateString)
  return 'Неизвестно'
}

// ДОБАВЛЕНА защита от отрицательной разницы (дата в будущем)
if (diffMs < -60000) {  // > 1 минуты в будущем
  console.warn('Дата в будущем:', dateString)
  return 'Только что'
}

// ДОБАВЛЕНА обработка небольших отрицательных разниц (до 1 минуты)
if (diffMs < 0) {
  return 'Только что'
}
```

## 📝 Технические детали

### Формат ISO 8601

| Формат | Пример | Значение |
|--------|--------|----------|
| Без суффикса | `2026-04-01T10:00:00` | Неясно (локальное или UTC?) |
| С 'Z' | `2026-04-01T10:00:00Z` | UTC (Zero timezone offset) |
| С offset | `2026-04-01T10:00:00+05:00` | UTC+5 (Екатеринбург) |

### JavaScript Date

```javascript
// Без 'Z' — браузер считает локальным временем
new Date('2026-04-01T10:00:00')  // Локальное время

// С 'Z' — браузер преобразует из UTC
new Date('2026-04-01T10:00:00Z') // UTC время, преобразованное в локальное
```
