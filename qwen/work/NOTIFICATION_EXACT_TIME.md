# 🕐 Отображение точного времени уведомлений

## 📅 Дата: 1 апреля 2026

## ✅ Что изменено

Теперь уведомления отображаются **с точным временем** (до минут) + относительное время для свежих.

---

## 📊 Форматы отображения

| Возраст уведомления | Формат | Пример |
|---------------------|--------|--------|
| < 1 минуты | Относительное | "Только что" |
| < 1 часа | Относительное | "5 мин. назад" |
| < 24 часов | Относительное | "3 ч. назад" |
| < 7 дней | Относительное | "2 дн. назад" |
| ≥ 7 дней | **Точное время** | "1 апр. 2026, 14:35" |

---

## 🔧 Изменения

### 1. Backend — явное указание UTC

**Файл:** `app/api/v1/routes/notification.py`

```python
from datetime import timezone

# Возвращаем время с явным указанием UTC
"created_at": notification.created_at.replace(tzinfo=timezone.utc).isoformat()
```

**Результат:** `2026-04-01T10:00:00+00:00` — браузер правильно конвертирует в локальное время

---

### 2. Frontend — точное время для старых уведомлений

**Файлы:**
- `src/views/Notifications.vue`
- `src/components/NotificationsDropdown.vue`

**Изменения:**

```javascript
// Для старых уведомлений (≥ 7 дней) показываем точное время
return date.toLocaleDateString('ru-RU', { 
  day: 'numeric', 
  month: 'long',
  year: 'numeric',
  hour: '2-digit',
  minute: '2-digit'
})
```

**Результат:** "1 апр. 2026, 14:35"

## 📚 Технические детали

### Формат ISO 8601 с timezone

| Формат | Пример | Как интерпретирует браузер |
|--------|--------|---------------------------|
| Без timezone | `2026-04-01T10:00:00` | Как локальное время ❌ |
| С 'Z' | `2026-04-01T10:00:00Z` | UTC, конвертирует в локальное ✅ |
| С offset | `2026-04-01T10:00:00+00:00` | UTC, конвертирует в локальное ✅ |

### JavaScript Date

```javascript
// Браузер автоматически конвертирует UTC в локальное время
const date = new Date('2026-04-01T10:00:00+00:00')

// Для пользователя в UTC+5 (Екатеринбург)
// date.toLocaleString() → "1 апр. 2026, 15:00"
```
