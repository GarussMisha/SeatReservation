# Схема базы данных - Система бронирования рабочих мест

## Обзор
Система управления бронированием рабочих мест с разделением на сущности: статусы, аккаунты пользователей, помещения, рабочие места и бронирования.

## Диаграмма связей
```
Status (1) ←───→ (M) Account
Status (1) ←───→ (M) Room
Room (1) ←───→ (M) Workspace
Status (1) ←───→ (M) Booking
Account (1) ←───→ (M) Booking
Workspace (1) ←───→ (M) Booking
```

## Детальная структура таблиц

### 1. statuses (Универсальные статусы)
**Описание:** Справочная таблица статусов для всех сущностей системы
| Поле | Тип | Ограничения | Описание |
|------|-----|-------------|----------|
| id | Integer | PK, AutoIncrement | Уникальный идентификатор |
| created_at | DateTime | NOT NULL | Дата создания записи |
| name | String(50) | UNIQUE, NOT NULL | Название статуса |
| description | String(255) | NULL | Описание статуса |

**Связи:**
- accounts (M:1) - аккаунты с данным статусом
- rooms (M:1) - помещения с данным статусом
- bookings (M:1) - бронирования с данным статусом

### 2. accounts (Аккаунты пользователей)
**Описание:** Учетные записи для аутентификации и доступа к системе (включают персональные данные)
| Поле | Тип | Ограничения | Описание |
|------|-----|-------------|----------|
| id | Integer | PK, AutoIncrement | Уникальный идентификатор |
| created_at | DateTime | NOT NULL | Дата создания записи |
| login | String(100) | UNIQUE, NOT NULL | Логин для входа |
| password_hash | String(255) | NOT NULL | Хеш пароля |
| password_set_at | DateTime | NOT NULL | Дата установки пароля |
| is_admin | Boolean | NOT NULL, DEFAULT FALSE | Признак администратора |
| status_id | Integer | FK→statuses.id, NOT NULL | Статус аккаунта |
| first_name | String(100) | NOT NULL | Имя пользователя |
| last_name | String(100) | NOT NULL | Фамилия пользователя |
| email | String(255) | NULL | Email пользователя |
| middle_name | String(100) | NULL | Отчество пользователя |
| birth_date | Date | NULL | Дата рождения пользователя |
| phone | String(20) | NULL | Телефон пользователя |

**Связи:**
- status (M:1) - статус аккаунта
- bookings (M:1) - бронирования аккаунта

**Ограничения:**
- UNIQUE(login) - уникальный логин

### 3. rooms (Помещения/Комнаты)
**Описание:** Физические локации компании с рабочими местами
| Поле | Тип | Ограничения | Описание |
|------|-----|-------------|----------|
| id | Integer | PK, AutoIncrement | Уникальный идентификатор |
| created_at | DateTime | NOT NULL | Дата создания записи |
| name | String(200) | UNIQUE, NOT NULL | Название помещения |
| address | String(500) | NULL | Адрес помещения |
| description | String(1000) | NULL | Описание помещения |
| status_id | Integer | FK→statuses.id, NOT NULL | Статус помещения |

**Связи:**
- status (M:1) - статус помещения
- workspaces (1:M) - рабочие места в помещении

### 4. workspaces (Рабочие места)
**Описание:** Отдельные рабочие места, доступные для бронирования
| Поле | Тип | Ограничения | Описание |
|------|-----|-------------|----------|
| id | Integer | PK, AutoIncrement | Уникальный идентификатор |
| created_at | DateTime | NOT NULL | Дата создания записи |
| name | String(200) | NOT NULL | Название рабочего места |
| is_active | Boolean | NOT NULL, DEFAULT TRUE | Признак активности |
| room_id | Integer | FK→rooms.id, NOT NULL | Помещение рабочего места |

**Связи:**
- room (M:1) - помещение рабочего места
- bookings (1:M) - бронирования места

**Ограничения:**
- UNIQUE(name, room_id) - уникальное название в рамках помещения

### 5. bookings (Бронирования)
**Описание:** Записи о бронировании рабочих мест на определенные даты
| Поле | Тип | Ограничения | Описание |
|------|-----|-------------|----------|
| id | Integer | PK, AutoIncrement | Уникальный идентификатор |
| created_at | DateTime | NOT NULL | Дата создания записи |
| booking_date | Date | NOT NULL | Дата бронирования |
| status_id | Integer | FK→statuses.id, NOT NULL | Статус бронирования |
| account_id | Integer | FK→accounts.id, NOT NULL | Кто забронировал |
| workspace_id | Integer | FK→workspaces.id, NOT NULL | Какое место забронировано |

**Связи:**
- account (M:1) - аккаунт забронировавшего
- workspace (M:1) - забронированное место
- status (M:1) - статус бронирования

## Общие принципы

### Базовый класс BaseModel
Все таблицы наследуются от BaseModel и автоматически имеют:
- `id` (Integer, PK, AutoIncrement)
- `created_at` (DateTime, NOT NULL)

### Каскадные операции
Все связи настроены с `cascade="all, delete-orphan"` для автоматического удаления связанных записей.

### Универсальная система статусов
Status - центральная таблица для управления состоянием всех сущностей в системе, что обеспечивает:
- Консистентность статусов
- Возможность централизованного управления
- Гибкость в добавлении новых статусов

### Безопасность данных
- Хеширование паролей (password_hash)
- Уникальные логины
- Обязательные внешние ключи
- Проверка целостности данных

## Примеры использования статусов

### Для Account (Аккаунт):
- "active" - активный аккаунт
- "inactive" - неактивный аккаунт
- "blocked" - заблокированный аккаунт
- "pending" - ожидающий подтверждения
- "on_leave" - в отпуске
- "sick" - на больничном
- "terminated" - уволенный

### Для Room (Помещение):
- "available" - доступно
- "maintenance" - на обслуживании
- "unavailable" - недоступно

### Для Booking (Бронирование):
- "confirmed" - подтверждено
- "pending" - ожидает подтверждения
- "cancelled" - отменено
- "completed" - завершено
